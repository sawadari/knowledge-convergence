# 例：材質変更が hold になる

## 状況

製品チームが、コスト削減のために材質変更を提案します。

提案には次が含まれます。

- 推定コスト削減効果
- 初期的な製造性レビュー
- 概算スケジュール影響

しかし、次が不足しています。

- 安全性の妥当性確認
- ライフサイクル耐久性の根拠
- サプライヤ影響レビュー
- 顧客受容性の分析

## 判断候補

```text
コスト削減のため、材質Aを材質Bへ変更する。
```

## 知識収束チェック

| 次元 | 状態 |
|---|---|
| 意味 | 明確 |
| 文脈 | 対象製品ラインと対象部品は指定済み |
| 根拠 | コスト根拠はあるが、安全根拠が不足 |
| 価値基準 | コストは考慮済みだが、安全性と耐久性が不完全 |
| 責任 | 技術オーナーはいるが、サプライヤレビュー責任者が不足 |
| 履歴 | 初期提案は記録済み |
| ドメイン妥当性 | 不十分 |

## 結果

```text
outcome = hold
reason = validation evidence and supplier review are missing
next_action = request safety validation and supplier impact review
```

## 教訓

提案は経済的に魅力的でも、対象ドメインで妥当とは限りません。実行条件を満たさない場合、正しい分岐は hold です。
