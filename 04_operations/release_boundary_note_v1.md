# リリース境界メモ

## 何が紛らわしかったか
元パッケージでは、`version: 3.0 / 3.1` の作業版表記と、`canonical release v1` の凍結版表記が同居していました。  
内部運用としては合理的ですが、外から読むと「どれが公開上の正本か」が見えにくい状態でした。

## この再パッケージ版での整理
- **公開上の理論コア** は `v1`
- **研究系譜の追跡ラベル** は `v3.1`
- その境界を `01_core/` と `04_operations/` で分けました

## どう読むべきか
- 理論の正本として読む: `01_core/`
- 研究運用・引き継ぎとして読む: `04_operations/`
- 元のまま検証する: `99_archive/original_flat_package/`

## 重要な保全方針
この再パッケージ版は、**元ファイルを削除・改変して歴史を塗り替えるものではありません**。  
元のフラット構成は `99_archive/original_flat_package/` にそのまま保存し、読みやすい配布境界だけを作り直しています。
