# 1ページ概要

知識収束学は、生成物を判断可能な知識状態へ変換するためのフレームワークです。

## 中核文

生成物は、そのまま知識ではありません。

生成物は、次と接続されたときに実務で使える知識になります。

- 意味
- 文脈
- 根拠
- 価値基準
- 責任
- 履歴
- ドメイン妥当性

## 中核モデル

```text
K = (G, C, E, V, R, H)
```

## 三層

1. 認識収束: 内容として説明できるか
2. 統治収束: 組織が責任を持って扱えるか
3. ドメイン妥当性収束: 対象ドメインと意図した用途に対して妥当か

## 主な適用先

- AIエージェント統治
- システムズエンジニアリング
- 要求・判断管理
- 検証と妥当性確認
- 組織を含むAIワークフロー
- 適合検査とLint

## 重要な区別

AIコーディングは、実装速度を上げます。

知識収束学は、何を実装するか、なぜ実装するか、誰が責任を持つか、どう妥当性確認するかの品質と説明責任を上げます。
