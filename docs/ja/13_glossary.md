# 用語集

## Agent execution envelope / エージェント実行包絡

AIエージェントが何をしてよいかを定義する境界です。エージェントID、オーナー、権限範囲、ツール範囲、データアクセス範囲、レビューゲート、監査ログ、停止条件、ロールバック経路を含みます。

## Candidate knowledge / 知識候補

実務で使える知識になり得るが、まだ責任ある行動には使えない情報、出力、主張、モデル、文書、結果です。

## Conformance / 適合

知識状態が、定義されたschema、profile、意味規則を満たす程度です。

## Decision-ready knowledge / 判断可能な知識

意味、文脈、根拠、価値基準、責任、履歴、ドメイン妥当性が十分に接続され、組織が責任ある分岐を選べる知識状態です。

## Domain validity convergence / ドメイン妥当性収束

知識状態が、対象ドメイン、運用環境、意図した用途に対して十分妥当かを確認する収束層です。

## Epistemic convergence / 認識収束

内容が、意味、文脈、根拠、前提、不確実性を含めて説明可能かを確認する収束層です。

## Governance convergence / 統治収束

組織が、その状態を責任を持って判断、承認、保留、棄却、実行、エスカレーション、ロールバックできるかを確認する収束層です。

## Hold / 保留

理由を明示して実行を止める正当な分岐です。hold は失敗ではありません。制御された状態です。

## Knowledge state / 知識状態

意味、文脈、根拠、価値基準、責任、履歴を接続した構造化状態です。

## Meaningful human oversight / 意味ある人間監督

人間が実際に理解し、レビューし、停止し、エスカレーションし、修正できる状態です。形式的な承認チェックボックスは、意味ある監督ではありません。

## Organization topology / 組織トポロジー

知識状態の中で、役割、意思決定権、責任、境界、レビュー能力、エスカレーション経路、利益相反を表現したものです。

## SE System / SEシステム

知識収束学に基づく、システムズエンジニアリングの知識・意思決定基盤です。コード前の判断、要求、V&V、人間の役割、組織構造、AI委任、変更影響を管理します。

## Validation / 妥当性確認

システム、判断、要求が、意図した利用、ステークホルダニーズ、運用文脈、事業価値を満たすかを確認することです。

## Verification / 検証

システム、判断、成果物が、指定要求または定義された基準を満たすかを確認することです。
