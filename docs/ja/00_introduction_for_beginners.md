# 初学者向け導入

知識収束学は、情報を、人間・組織・AIシステムが責任を持って使える状態へ変換するための考え方です。

中心となる考え方は単純です。

> 生成物は、そのまま知識ではない。

AIの回答、人間のコメント、議事録、テスト結果、要求案は有用かもしれません。しかし、それらは自動的に実務で使える知識にはなりません。文脈、根拠、価値基準、責任、履歴、ドメイン妥当性と結びついたときに、行動に使える知識になります。

## なぜ重要か

AIシステムは、大量の文章、コード変更、テストケース、計画、図、要約を生成できます。これは有用ですが、新しいボトルネックを生みます。

ボトルネックは、生成だけではありません。判断です。

チームは、依然として次を決める必要があります。

- この出力は十分に正しいか
- どこで成り立つか
- どの根拠に支えられているか
- 誰が責任を持つか
- どのリスクがあるか
- 実行してよいか
- 保留、棄却、エスカレーション、ロールバックすべきか

知識収束学は、この過程を扱う語彙と構造を提供します。

## 収束とは何か

収束とは、全員の意見を一つに潰すことではありません。

収束とは、責任を持って次の分岐を選べる状態に到達することです。

- 実行する
- 保留する
- 棄却する
- エスカレーションする
- 再オープンする
- ロールバックする

正当な hold は失敗ではありません。理由が明示された制御状態です。

## 例

AIエージェントがコード変更を提案したとします。

コードはもっともらしく見えます。しかし、次が欠けています。

- 要求が承認されていない
- 妥当性確認シナリオがない
- ロールバック経路が定義されていない
- ドメイン専門家がレビューしていない

知識収束学では、この状態を実行可能とはみなしません。正しい結果は次のようなものになります。

```text
outcome = hold
reason = missing validation and authority envelope
```

## どこで使えるか

知識収束学は、次で使えます。

- AIエージェント統治
- システムズエンジニアリング
- 要求工学
- 設計判断管理
- 検証と妥当性確認
- 組織的意思決定
- 業務自動化
- 変更影響分析
