# 問題設定：生成物は判断可能な知識ではない

AIは生成コストを下げます。しかし、判断、責任、妥当性確認、統治の必要性は消えません。

このリポジトリが扱うのは、次の間にあるギャップです。

```text
生成物
```

と、

```text
責任を持てる、根拠付きの、対象ドメインで妥当な知識
```

の間にあるギャップです。

## 1. AIは生成コストを下げる

AIシステムは、次を生成できます。

- 文章
- コード
- テスト
- 要求
- 計画
- 要約
- 図
- ワークフロー操作
- ツール呼び出し

これにより、開発の経済性は変わります。成果物を作ることは容易になります。

## 2. しかし判断コストは残る

組織は、依然として次の問いに答える必要があります。

- 出力は十分に正しいか
- どの根拠に支えられているか
- どの前提に依存するか
- どのドメイン制約が適用されるか
- 誰が承認できるか
- 誰が実行できるか
- どう妥当性確認するか
- 失敗したら何が起きるか
- ロールバックできるか

これらは、成果物をさらに生成するだけでは解決しません。

## 3. 新しいボトルネック

生成が安くなると、上流の弱い判断はより危険になります。

組織は、間違った文書、間違ったコード、間違ったテスト、間違った自動化を、以前より速く作れるようになります。

ボトルネックは、成果物生産から知識統治へ移ります。

## 4. 知識収束学が扱うこと

知識収束学は、候補出力を判断可能な知識状態へ変換する方法を定義します。

必要なのは、次の接続です。

- 意味構造
- 文脈
- 根拠
- 価値基準
- 責任と権限
- 履歴
- ドメイン妥当性

## 5. AIとSEにとっての重要性

AI研究にとって、知識収束学はモデル出力品質と責任ある利用を接続します。

システムズエンジニアリングにとって、知識収束学はステークホルダニーズ、設計判断、要求、検証、妥当性確認、AI委任、変更管理を接続します。
