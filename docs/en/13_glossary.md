# Glossary

## Agent execution envelope

A bounded definition of what an AI agent is allowed to do. It includes identity, owner, authority scope, tool scope, data access scope, review gate, audit log, stop condition, and rollback path.

## Candidate knowledge

Information, output, claim, model, document, or result that may become usable knowledge but is not ready for accountable action yet.

## Conformance

The degree to which a knowledge state satisfies defined schemas, profiles, and semantic rules.

## Decision-ready knowledge

A knowledge state that has enough meaning, context, evidence, value criteria, responsibility, history, and domain validity for an organization to select an accountable branch.

## Domain validity convergence

The convergence layer that checks whether a knowledge state is valid enough for the target domain, operational environment, and intended use.

## Epistemic convergence

The convergence layer that checks whether the content is explainable with meaning, context, evidence, assumptions, and uncertainty.

## Governance convergence

The convergence layer that checks whether an organization can responsibly decide, approve, hold, reject, execute, escalate, or rollback the state.

## Hold

A valid outcome branch where execution is blocked for a stated reason. Hold is not failure. It is a controlled state.

## Knowledge state

A structured state connecting meaning, context, evidence, value criteria, responsibility, and history.

## Meaningful human oversight

A state where a human can actually understand, review, stop, escalate, or correct an action. A symbolic approval checkbox is not meaningful oversight.

## Organization topology

The representation of roles, decision rights, responsibility, boundaries, review capacity, escalation paths, and conflicts of interest within the knowledge state.

## SE System

A Systems Engineering knowledge and decision infrastructure based on Knowledge Convergence. It manages pre-code decisions, requirements, V&V, human roles, organization structure, AI delegation, and change impact.

## Validation

A check that the system, decision, or requirement satisfies intended use, stakeholder need, operational context, or business value.

## Verification

A check that the system, decision, or artifact satisfies specified requirements or defined criteria.
