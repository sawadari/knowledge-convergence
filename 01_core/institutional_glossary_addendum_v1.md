# 制度運用発展 用語集 addendum

- **epistemic justification packet**: claim や evidence candidate を、誰の証言か、どの根拠か、どの範囲まで継承可能かで包む追加 packet。
- **speaker warrant profile**: 話者・情報源がどのタイプの warrant を持つかを表す型。真理保証ではない。
- **defeater**: 既存の support を弱める、範囲外化する、無効化する事情。
- **epistemic transfer**: 証言や報告から、聞き手側の claim support へ justification がどこまで移るかという関係。
- **aggregation rule profile**: 複数人レビューをどの方式で一つの結果へまとめるかを表す packet。
- **premise-based / conclusion-based**: 各前提を集約してから結論を出すか、各人の結論票を直接集約するかの違い。
- **minority report**: 少数意見や dissent を、最終 decision と別に retain する記録。
- **lifecycle gate packet**: decision outcome を、baseline・gate・成果物・移行条件へ接続する packet。
- **transition package**: release / deployment / handoff / rollback に必要な成果物束。
- **ops feedback event**: 運用中の観測・障害・逸脱を、再評価や rollback へ戻すイベント。
- **human oversight packet**: 人の役割、説明範囲、停止権、訓練完了、手動代替を束ねる packet。
- **manual fallback**: 自動経路が不適なときに安全に戻れる手動運用経路。
